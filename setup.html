Setup

Setting up eCommerce MVC is very simple. After extracting files from zip files, you can open project solution file in Visual Studio. We recommend to use Visual Studio 2019.

Database

eCommerce MVC uses Entity Framework Code first approach to create database. You don't need any SQL scripts to create database and its tables and columns. Follow below steps to setup database. Just change the Connection String in Web.config according to to your database server instance. eCommerce MVC will automatically create database tables and relationships with the help of entity framework.

For Local Host

When you are trying to open eCommerce MVC in your local development environemnt.

  1. Open Command Prompt by pressing Win+R and then type cmd in run window. Command Prompt will be opened. Get the name of SQL server instances by typing  sqllocaldb i  in your command prompt.
  2. Open eCommerce MVC in Visual Studio.
  3. Open web.config file from eCommerce MVC project and search for eCommerceConnectionString-OK in connectionStrings tag.
  4. Change  MSSQLLocalDB  with your preferred server instance name.
  5. Open Tools>Nuget Packager Manager>Nuget Package Manager Console window from Menu in Visual Studio
  6. Run  Update-Database  command in this window. It is Entity Framework Migration command. It'll take a moment to create database and relationships.
  7. Open Quizbee in browser by pressing Ctrl+F5.
For Remote/Release

When you are trying to publish eCommerce MVC with remote database.

  1. Get the connection string for your remote database server. You can create a new empty database or choose an existing one. If you select an existing database with few tables, you might need to run migration commands to update database.
  2. Open eCommerce MVC in Visual Studio.
  3. Open web.config file from eCommerce MVC project and search for eCommerceConnectionString-OK in connectionStrings tag.
  4. Change connection string value to your connection string from step 1.
  5. Publish eCommerce MVC to your server. We recommend to use Microsoft Azure or Amazon AWS but you can host it on any windows server.
  6. Open your published site in the browser. It'll take a moment when loading for the first time because Entity Framework is creating database and its entities and seeding the initial data in the background.
Remember NOT to update the name of connection string. It's name must remain eCommerceConnectionString-OK. This is used to identify connection string in the eCommerceContext class.
Other Database Providers

The default database server configured for eCommerce MVC is Microsoft SQL Server. However if you intend to use any other database providers with eCommerce MVC, you can do so easily without making a lot of changes to database logics. This is because eCommerce MVC uses Entity Framework and Entity Framework supports a number of database providers. Below are the links for setting most common database providers with entity framework.

  1. PostgreSQL
  2. MySql
  3. Oracle Data Provider
  4. SQLite
  5. IBM Data Server (DB2)

Default Roles and User

eCommerce MVC also seed default roles and user data with the help of Entity Framework when database is first created. It is defined in eCommerceDBInitializer.cs class which exists in eCommerce.Data project.

  1. Administrator
  2. Moderator
  3. User
Administrator and User Role should not be deleted or renamed beacuse these are being used in different sections of code for Authorization to Dashboard Panel.

eCommerce MVC also has a default Adminstrator user with the following credentials.

  1. Email: admin@email.com
  2. Username: admin
  3. Password: admin123
Remember to change these admin credentials after deploying eCommerce MVC.
