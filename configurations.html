Configurations

We implemented eCommerce MVC in such a way that anyone can configure it depending on their needs.

Application Info and Social Links Configuration

You can configure one time configurations such as Application Name, Company Address (shown at the bottom), Phone Number, Mobile Number, Facebook Page URL, Twitter URL, Twitter Username, WhatsApp Number, Instagram URL, YouTube URL and LinkedInURL in the appSettings node in web.config file. Social Links everywhere in the website uses configurations from this file.

System Configurations

eCommerce MVC allows you to update the following system configurations from Configurations screen in Dashboard panel. Only users with Administrator role will be able to change these configurations.

  1. CurrencySymbol
    Default Value: $
    Description: This currency symbol is shown with prices on website and invoices.
    Available Options: $, €, etc
    If you plan to use any other currency than US$ with Credit Card payment option, you will need to apply for enabling that currency payments from Authorize .Net payment gateway.
  2. PriceCurrencyPosition
    Default Value: {price}{currency}
    Description: This configuration will set price and currency position with respect to each other accross the website. {price} will be replaced with the price value and {currency} will be replaced by configured currency symbol.
    Available Options: {price}{currency}, {currency}{price}
  3. EnableCreditCardPayment
    Default Value: true
    Description: Set value to true or 1 to enable Credit card payments or set value to 0 or false to disable credit card payments.
    Available Options: true, 1 (to set as true), false, 0 (to set as false)
  4. EnableCashOnDeliveryMethod
    Default Value: true
    Description: Set value to true or 1 to enable Cash on Delivery Method or set value to 0 or false to disable Cash on Delivery Method.
    Available Options: true, 1 (to set as true), false, 0 (to set as false)
  5. FlatDeliveryCharges
    Default Value: 0
    Description: Set the value for Delivery Charges. This is flat delivery charges rate. This will be add to the final amount and displayed to user on the Checkout page.
    Available Options: 0 to any number.
    eCommerce MVC does not support destination based Delivery charges. You can only configure fixed flat delivery charges.
  6. FrontendRecordsSizePerPage
    Default Value: 6
    Description: This is the number of products displayed on a single page to the users on home page or when a user search by product category.
    Available Options: any number.
    The higher this number is, the more time it will take to load the page due to product pictures. Keep the value modest between 6 to 12 preferably.
  7. FeaturedRecordsSizePerPage
    Default Value: 3
    Description: This is the number of Featured products displayed on a single page to the users on product details page.
    Available Options: any number.
    Ideally the value should be between 2 to 5. This is because the usual length of product description section will be less than the length of featured products section.
  8. DashboardRecordsSizePerPage
    Default Value: 10
    Description: This is the number of records displayed on listing pages of eCommerce MVC's Dashboard Panel.
    Available Options: any number.
    The higher this number is, the more time it will take to load the page. Keep the value modest between 5 to 20 preferably.
  9. Slider1
    Default Value: site/slider/slider1.png
    Description: This is the first image in the slider on home page. You should preferably upload image of size 1200x300 pixels.
    Available Options: Upload any picture.
  10. Slider2
    Default Value: site/slider/slider2.png
    Description: This is the second image in the slider on home page. You should preferably upload image of size 1200x300 pixels.
    Available Options: Upload any picture.
  11. Slider3
    Default Value: site/slider/slider3.png
    Description: This is the third image in the slider on home page. You should preferably upload image of size 1200x300 pixels.
    Available Options: Upload any picture.
  12. Slider4
    Default Value: site/slider/slider4.png
    Description: This is the fourth image in the slider on home page. You should preferably upload image of size 1200x300 pixels.
    Available Options: Upload any picture.

Authorize .Net Payment API Gateway Configuration

eCommerce MVC provides Credit Card payment option. If you set the EnableCreditCardPayment to true in Systems Configuration, you must have to setup your account with Authorize .Net as described in the video and change the following keys in the appSettings node in web.config file. You can check the implementation of Authorize .Net payment API in AuthorizeNetHelper class in Helpers folder in main Code folder in eCommerceMVC module.

Set the value of AuthorizeNet_SandBox to false so eCommerce MVC can process real transactions. If you need to test Credit Card payments with Authorize .Net Sandbox account on eCommerce MVC, set value of AuthorizeNet_SandBox to true.
